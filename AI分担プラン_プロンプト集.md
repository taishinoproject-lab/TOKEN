# 訪剣（TOKEN）データ整備 AI分担プラン & プロンプト集

作成日: 2026-07-16

## 0. 全体の流れ

```
[調査フェーズ]           [整形フェーズ]        [反映フェーズ]         [画像フェーズ]        [検証]
Gemini Deep Research → 出力JSONを人間が確認 → Antigravityがコード反映 → Claude Sonnetが画像 → Sonnetで最終確認
（クレジット消費ほぼゼロ）                     （Claudeクレジット温存）  （Chrome連携が必須）
```

原則: **「調べる」は無料枠のAI、「コードを書き換える」はAntigravity、「ブラウザから画像を取ってくる」だけClaude Sonnet** に限定します。Opus/Fableは通常作業には使いません（トラブル時の切り札）。

## 1. モデル使い分けの根拠

| モデル | 得意なこと | このプロジェクトでの役割 | 使わない理由/注意 |
|---|---|---|---|
| Gemini (Deep Research) | Web横断の情報収集・出典付きレポート | イベント調査、刀剣データ調査 | コードは書かせない（形式が崩れやすい） |
| Google Antigravity | リポジトリを見ながらのコード編集 | 調査結果のTSファイルへの反映、機械的な修正 | Chrome連携不可 → 画像作業は不可（実証済み） |
| Claude Sonnet (Cowork) | Claude in Chromeでの画像取得＋実装＋トリミング | 画像の差し替え・新規取得（昨日成功した手法） | ここにだけクレジットを使う |
| Claude Opus / Fable | 複雑な判断・トラブル対応 | Sonnetが失敗した場合のみ投入 | クレジット消費が大きい |

コスト構造の補足: Claudeのクレジット消費は「ツール呼び出し回数×コンテキスト量」でほぼ決まります。画像作業はChrome操作の往復が多いので、**1セッションに詰め込みすぎず、3〜4振りずつのバッチ**にするのが安全です（コンテキストが伸びると1呼び出しあたりの消費も増えるため）。

### T3: 出展刀剣の詳細データ調査【Gemini Deep Research】

```
あなたは日本刀の学術データ整備の担当者です。以下の刀剣について、正確な基礎データを調査してください。

【調査対象の刀剣リスト】
（← ここにT1・T2で判明した「アプリ未収録の刀剣」を貼り付ける。国宝を最優先、次に重要文化財）

【各刀剣について調べる項目】※アプリのデータ構造に対応
- name: 正式名称（例: 太刀 銘三条（名物三日月宗近））
- reading: 号や通称の読み（ひらがな）
- designation: 指定区分（国宝/重要文化財/重要美術品/特別重要刀剣/御物/未指定）
- category: 種別（太刀/打刀/脇指/短刀/大太刀/薙刀など）
- dimensions: 刃長・反り（cm単位、例: 刃長80.0cm 反り2.7cm）
- era: 時代（例: 平安時代、鎌倉時代・元徳2年(1330)）
- smith: 刀工名
- school: 流派（例: 山城国粟田口派）
- current_holder: 現在の所蔵者・所蔵館
- location: 所蔵館の所在地（都道府県・市区町村）
- provenance: 伝来（→でつなぐ。例: 足利将軍家→豊臣秀吉→徳川家）
- story: 号の由来や逸話を1〜2文で
- media_appearance: 刀剣乱舞に登場するか（登場する場合「刀剣乱舞」、しなければ「なし」）
- image_source_url: ColBase（https://colbase.nich.go.jp）に画像があればその作品ページURL。なければe国宝・文化遺産オンライン・所蔵館公式のURL。どこにもなければ「不明」

【ルール】
- 出典は所蔵館公式サイト・ColBase・e国宝・文化遺産オンライン・文化庁国指定文化財等データベースを優先
- 個人ブログやゲームwikiの数値をそのまま使わない（参考にした場合は要確認と明記）
- 不明な項目は「不明」とする。絶対に推測で数値を書かない
- 情報の確度を「事実」（公式出典で確認済み）/「推定」（二次情報のみ）で分類する

【出力形式】
以下のJSON配列で出力してください。
[
  {
    "name": "...", "reading": "...", "designation": "...", "category": "...",
    "dimensions": "...", "era": "...", "smith": "...", "school": "...",
    "current_holder": "...", "location": "...", "provenance": "...",
    "story": "...", "media_appearance": "...", "image_source_url": "...",
    "confidence": "事実 or 推定",
    "sources": ["出典URL"]
  }
]
```

## 5. 運用のコツ

- **バッチを小さく**: Claudeの画像作業は1セッション3〜4振り。セッションが長くなったら新しいチャットで続きを頼む（コンテキストが短いほどクレジット効率が良い）
- **Geminiの出力は目視確認してから渡す**: T3→T4の受け渡しが品質の要。特に刃長などの数値と指定区分は、1件でも怪しければその刀だけ「不明」に落とす方が安全
- **Antigravityにブラウザ作業をさせない**: Chrome連携が効かないことは実証済みなので、画像・チラシ関連は必ずClaude側に回す
- **Opus/Fableの出番**: Sonnetが画像のトリミングや実装で2回以上失敗したときだけ、同じプロンプトで引き継ぐ
- **著作権の注意**: ColBase・e国宝は出典明記で利用可。館の公式チラシは広報目的の利用が黙認されることが多いものの、アプリを一般公開する際は利用条件の確認を推奨（現段階のデモ用途とは区別する）
